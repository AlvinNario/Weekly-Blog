Week 1 Of On-The-Job Training For StraStan Solutions Corp.

February 4 - 6, 2025

The first day covered AWS-CDK fundamentals, including setting up projects, understanding key concepts, and hands-on introductory labs. Participants worked through basic infrastructure as code examples, debugging challenges, and explored best practices for managing cloud resources. The afternoon focused on advanced labs, troubleshooting common issues, and an interactive Q&A session to reinforce learning. The second day transitioned into hands-on AWS service implementation. Participants showcased their To-Do App projects and dived into AWS services such as Cognito for authentication, Lambda for serverless computing, DynamoDB for NoSQL databases, and SQS for message queuing. The day concluded with a practical lab, where attendees applied their learning to build and deploy applications using AWS-CDK.

As part of our onboarding and training at StraStan Solutions Corps, our team has been assigned to work on API gateway endpoints and their integration with AWS Lambda functions, DynamoDB tables, and other AWS services. Our primary focus this week is on two major modules: Marketplace and Cards.

Understanding the Task
The goal is to manage categories and sections in the Marketplace module while handling schema imports and verification processes in the Cards module. These operations involve interacting with API Gateway, triggering Lambda functions, storing data in DynamoDB, and ensuring event-driven processing through SQS queues and EventBridge rules.

Breakdown of the Work Assigned

1. Marketplace Module:
  • Updating and deleting categories using PUT and DELETE API endpoints.
  • Assigning subcategories to sections via a POST request.
  • Ensuring proper logging and monitoring for category modifications.
  • Handling event-driven updates using CategoryUpdateEvent and CategoryDeleteEvent.
2. Cards Module:
  • Importing card schemas from CSV files via POST /admin/cardschemas/csv.
  • Implementing content verification through POST /admin/listings/verify.
  • Utilizing SchemaImportQueue and VerifyQueue for asynchronous processing.
  • Ensuring the smooth execution of SchemaImportEvent and VerifyEvent triggers.

Key Technologies & Tools Used

  • AWS Lambda for backend logic execution.
  • DynamoDB for NoSQL database storage.
  • API Gateway for managing endpoints.
  • SQS for message queuing and event-driven architecture.
  • Cognito for authentication and authorization.
  • EventBridge for event processing.

Our Approach for the Week

  • Day 1-2: Familiarize ourselves with AWS services used in this project.
  • Day 3: Study existing API documentation and understand request-response flows.
  • Day 4: Implement and test API integrations in a sandbox environment.
  • Day 5: Debugging, monitoring logs, and reviewing performance optimizations.
By the end of the week, we aim to gain a deeper understanding of API-driven architecture, event processing, and AWS cloud services. This assignment will provide hands-on experience crucial for our future tasks at StraStan Solutions Corps.
